Balls and Boxes#

So far we‚Äôve looked at two different counting problems: counting \(k\)-permutations and counting \(k\)-combinations. It turns out there‚Äôs a way of looking at these problems which shows an interesting way they are related, and lights the way towards understanding the problems we‚Äôre going to see over the rest of this chapter.

In the previous section, we saw how we can represent a combination as a binary string:

Suppose each character of this string corresponds to a box. Wherever a 0 appears, we leave the box empty. Wherever a 1 appears, we place a ball in the box. Each box represents an element of the original set, and placing a ball in the box represents selecting that element to be included in a combination.

The most important thing to notice about this analogy is this: the boxes are labelled, while the balls are unlabelled. Exactly which ball a box gets doesn‚Äôt matter, since the balls are indistinguishable. However, which box a ball is placed in does matter, since that dictates which element of the set is chosen. In this sense, the boxes are labelled with the elements of the set.

This setup gives us an alternative statement of the combination problem: there are k!n ways to place \(k\) unlabelled balls into \(n\) labelled boxes, with at most one ball in each box.

In isolation this is not particularly enlightening, but things get more interesting when we notice that we can tweak the parameters of the problem in three different ways. We can change whether the balls are labelled, whether the boxes are labelled, and how many balls are allowed in each box. This gives way to a whole family of counting problems, and we‚Äôll be categorising them all in the next chapter. For now, let‚Äôs see how changing just one part of the problem effects things.

Let‚Äôs say that, instead of being indistinguishable, we‚Äôre going to label our \(k\) balls with ‚ç≥k. Now, it does matter exactly which ball goes into which box:

With labelled balls, the problem no longer corresponds to counting \(k\)-combinations. Instead, it corresponds to counting \(k\)-permutations. If we take the label of ball as the index in a \(k\)-permutation, placing the balls in the boxes corresponds to choosing which element is placed at each index.

In later sections, we‚Äôre going to look at the other specific counting problems we can make by varying the labelling of both the balls and boxes. We‚Äôre also going to look at how controlling the number of balls which can go into each box affects the problems we make, which is going to require being more formal with our analogy.

Counting Functions#

Consider again the balls and boxes example we‚Äôve just seen. It involves assigning a box to each ball we have. In other words, each placement of balls is equivalent to a monadic function which takes a ball as input, and returns the box it is placed in as output. We‚Äôve been working with our fruit example until now, but to show how our problems can work for any set, we‚Äôre going to start using generic sets of inputs and outputs.
